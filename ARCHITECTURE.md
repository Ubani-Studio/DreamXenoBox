# DREAM XENO BOX — Stage 1: Architecture

## Design Manifesto

Dream Xeno Box is a 6-voice alien percussion instrument where each voice is a material body under pressure, not a drum category. Sound is generated by excitation striking a resonant body, passing through fracture and atmospheric stages, shaped by accumulated memory of past performance. Repetition changes the instrument. Silence restores it. The sequencer is an invocation engine, not a loop player.

The instrument must sound: heavy, alien, ritual, biomechanical, fractured, definitive.
It must be: playable, repeatable, ownable, performable, portable to hardware.

## Per-Voice Signal Flow

```
TRIGGER (bang + velocity)
    │
    ▼
┌─────────────┐
│   EXCITER    │  ← exciter_type, pitch, velocity, heat
│  (transient) │     Generates initial impulse energy
└──────┬──────┘
       │ short burst signal (< 20ms)
       ▼
┌─────────────┐
│    BODY      │  ← body_type, weight, stiffness, fatigue
│  (resonance) │     Shapes material character
└──────┬──────┘
       │ resonant signal (10-2000ms)
       ├────────────────────┐
       ▼                    ▼
┌─────────────┐      ┌─────────────┐
│  FRACTURE    │      │  HALO       │  ← mist, bloom
│  (distortion)│      │  (atmosphere)│     Parallel atmospheric layer
└──────┬──────┘      └──────┬──────┘
       │ ← scar, pressure         │
       ▼                           ▼
┌──────────────────────────────────────┐
│            MIX + VCA                  │
│  fracture_out * (1-halo_mix)         │
│  + halo_out * halo_mix               │
│  → envelope → output                 │
└──────────────────────────────────────┘
       │
       ▼
   VOICE OUTPUT (mono, panned at mixer level)
```

Key: Halo is parallel to fracture, not serial. This lets the atmospheric tail breathe independently of the destruction stage.

## Exciter Types (2 for prototype, 4 planned)

### 1. FM Impulse Click
- 2-operator FM: carrier = pitch frequency, modulator = carrier × ratio
- Modulation index starts high (8-20), decays to 0 in 1-5ms
- Produces sharp, spectrally rich click with controllable brightness
- `heat` state increases initial mod index → brighter attack when hot

```
carrier_freq = mtof(pitch)
mod_freq = carrier_freq * fm_ratio
mod_index_env = index_start * exp(-t * index_decay)
exciter = sin(carrier_phase + mod_index_env * sin(mod_phase))
```

### 2. Noise Burst
- Filtered white noise with fast exponential envelope
- Filter cutoff controlled by `pitch` and `heat`
- Envelope: 0.5-10ms attack, 1-20ms decay
- Produces textural, breathy transient

```
noise_filtered = biquad_bandpass(noise(), cutoff, Q)
exciter = noise_filtered * burst_envelope
```

### Planned (Stage 5):
3. Phase-Warped Chirp — rapid frequency sweep
4. Folded Sine Crack — sine through wavefolder at excitation

## Body Types (2 for prototype, 4 planned)

### 1. Inharmonic Resonator Bank
- 4 parallel resonant bandpass filters at non-integer frequency ratios
- Ratios: 1.0, 1.347, 1.891, 2.534 (tunable, these are bell-like defaults)
- Each resonator: biquad bandpass, Q controlled by `fatigue` state
- Higher fatigue → lower Q → deader sound
- `weight` macro scales all frequencies down (heavier) or up (lighter)
- `stiffness` state shifts ratios toward integer (more harmonic) or away (more inharmonic)

```
body = 0
for each resonator i:
    freq_i = base_freq * ratio[i] * (1 + stiffness * ratio_shift[i])
    Q_i = base_Q * (1 - fatigue * 0.7)
    body += biquad_bp(exciter, freq_i, Q_i) * gain[i]
```

### 2. Dispersive Comb
- Short comb filter (0.5-10ms delay) with frequency-dependent damping
- Simulates membrane/tube/skin resonance
- Delay time from `pitch`, feedback from `fatigue` (inverted: fatigue reduces feedback)
- One-pole lowpass in feedback loop → higher frequencies die faster
- `weight` scales delay time → heavier = longer delay = lower pitch

```
delay_samples = srate / mtof(pitch + weight_offset)
feedback_filtered = onepole(delay_read, damping)
delay_write = exciter + feedback_filtered * fb_amount * (1 - fatigue * 0.5)
body = delay_read
```

### Planned (Stage 5):
3. Modal Body — multiple mode frequencies from physical dimensions
4. Pressure Cavity — closed waveguide tube with nonlinear reflection

## Fracture Types (1 for prototype, 3 planned)

### 1. Asymmetric Wavefold
- Different fold thresholds for positive and negative half-cycles
- `scar` macro controls fold depth
- `pressure` state increases asymmetry → more lopsided harmonics
- Clean at scar=0, increasingly shattered as scar rises

```
// asymmetric fold with pressure-dependent bias
biased = signal + pressure * asymmetry_amount
folded = asin(sin(biased * fold_depth * pi)) / pi
// fold_depth = 1 + scar * 4  (1x to 5x folding)
```

### Planned (Stage 5):
2. Phase Tear — discontinuous phase jumps
3. Saturation Scar — tanh with memory (harder clip as pressure builds)

---

## Halo Types (1 for prototype, 3 planned)

### 1. Noise Veil
- Filtered noise envelope triggered by each hit
- Longer decay than exciter (50-500ms)
- Bandpass filtered, center frequency follows `pitch`
- `mist` macro controls level, `bloom` controls decay time
- Adds breathy atmospheric tail to each hit

```
halo_env = halo_env * (1 - 1/(bloom_decay_samples))  // exponential decay
on trigger: halo_env = velocity * mist
halo = biquad_bp(noise(), halo_freq, halo_Q) * halo_env
```

### Planned (Stage 5):
2. Comb Mist — very short comb with high feedback, noise-fed
3. Resonant Ghost — delayed filtered body signal with long tail

## Memory / Pressure State Model

All state variables are per-voice `History` objects in gen~. They persist between samples and between triggers.

### State Variables

| Variable | Range | Trigger Effect | Silence Effect | Sonic Impact |
|----------|-------|----------------|----------------|--------------|
| `pressure` | 0-1 | += velocity × sensitivity × (1-pressure) | *= (1 - decay_rate) per sample | fracture depth, pitch instability |
| `heat` | 0-1 | follows pressure with lag | decays faster than pressure | exciter brightness, filter cutoff |
| `fatigue` | 0-1 | += small amount per hit | decays very slowly | body damping (Q reduction), deader sound |
| `stiffness` | 0-1 | = f(fatigue, pressure) derived | follows fatigue | body ratio shift, less sustain |

### Update Rules (per sample)

```
// On trigger detection (trigger goes from 0 to non-zero):
pressure += velocity * pressure_sensitivity * (1.0 - pressure)
fatigue += 0.02 * (1.0 - fatigue)
hit_count += 1

// Every sample (always running):
pressure *= (1.0 - pressure_decay)     // pressure_decay ~ 0.00001-0.0001
heat += (pressure - heat) * heat_lag    // heat_lag ~ 0.0001 (follows pressure)
heat *= (1.0 - heat_decay)             // heat_decay ~ 0.00005
fatigue *= (1.0 - fatigue_decay)       // fatigue_decay ~ 0.000001 (very slow)
stiffness = fatigue * 0.6 + pressure * 0.4

// Derived modulations:
fracture_mod = scar + pressure * stress
brightness_mod = brightness + heat * 0.5
damping_mod = base_damping + fatigue * 0.7
pitch_wobble = noise_slow * pressure * drift * 2.0  // pitch instability
```

### Timing at 44100 Hz

| State | Full charge (0→1) | Full decay (1→0) | Musical meaning |
|-------|-------------------|-------------------|-----------------|
| pressure | ~5 dense hits | ~0.5-2 seconds | "the voice is being pushed" |
| heat | lags pressure by ~200ms | ~1 second | "the voice is glowing" |
| fatigue | ~50+ hits | ~10-30 seconds | "the voice is worn out" |
| stiffness | derived, no direct accumulation | follows fatigue/pressure | "the material has changed" |

## gen~ vs Max Split

### gen~ (per voice — audio rate, deterministic)
- Exciter generation (FM, noise)
- Body resonance (resonator bank, comb)
- Fracture processing (wavefold)
- Halo generation (noise veil)
- VCA envelope
- Memory state (all History variables)
- Trigger detection (edge detection on trigger input)
- All audio DSP

### Max (global — control rate, event-driven)
- Transport (metro, counter, swing)
- Sequencer (step data, pattern management)
- Macro controls (dials → parameter scaling → gen~ Params)
- Parameter lock dispatch (dict lookup → per-step overrides)
- Pressure law display (optional: show state visually)
- Mixer (gain~, pan)
- Master output routing
- Pattern/scene storage (dict)
- UI layout

### Why this split
- gen~ handles everything that must be sample-accurate and stateful
- Max handles everything event-based and UI-related
- No Max→gen~ communication in the audio path except Params (which update at control rate, fine for macros)
- Trigger enters gen~ as a Param that pulses from 0 to velocity, gen~ detects the edge internally

### Pressure Law (Max-side, per clock tick)

On each step:
1. Check if voice was triggered this step
2. If triggered: send velocity pulse to gen~ trigger Param
3. Track running hit density (hits in last N steps) per voice
4. Density feeds back into stress sensitivity (optional global pressure law)

The per-voice gen~ handles its own internal pressure accumulation. The sequencer pressure law is an additional macro-level system that can modulate parameters based on pattern density across the whole sequence.

## Sonic Reference Descriptions (not brand names)

What Dream Xeno Box should sound like, described in material terms:

1. **Pressurized metal struck inside a sealed chamber** — Mass voice with high weight, moderate scar
2. **Ceramic shattering in slow motion** — Shard voice with high scar, fast decay
3. **Taut wire vibrating against a bone surface** — Vein voice with resonator body, moderate heat
4. **Hollow dried gourd hit with increasing force** — Husk voice with comb body, rising pressure
5. **Earth cracking open, deep rumble with surface fracture** — Fault voice with long decay, high stress
6. **Ghost resonance hanging in fog after impact** — Halo voice with high mist, high bloom
7. **The 40th hit on the same surface — worn, scarred, different from the first** — any voice with accumulated fatigue
8. **Silence after intensity — the instrument cooling down** — pressure/heat/fatigue decaying

## CPU Risk Assessment

| Component | Per Voice | Notes |
|-----------|-----------|-------|
| FM exciter (2 oscs) | negligible | 2 sin() calls |
| Noise burst + biquad | negligible | noise + 5 multiplies |
| Resonator bank (4 biquads) | low | 4 × biquad = ~20 mul/add |
| Dispersive comb (delay + filter) | low | 1 delay read/write + onepole |
| Wavefold | negligible | asin(sin()) or polynomial approx |
| Noise veil | negligible | noise + biquad + env |
| State updates | negligible | 4 multiplies per sample |
| **Total per voice** | **low** | ~50 operations/sample |
| **6 voices** | **low-moderate** | ~300 operations/sample |

No FFT. No convolution. No heavy spectral processing. This should run well under 10% CPU on any modern machine. The expensive stuff (encoder bus, spectral processing) is excluded from the core architecture and can be added as a send effect later.

## RNBO / Hardware Portability Notes

This architecture is designed for future portability:
- All DSP in gen~ (RNBO-compatible)
- No Max-specific audio objects in the voice path
- State is self-contained per voice (no cross-voice audio dependencies)
- Parameter interface is flat (13 Params, 1 outlet)
- Sequencer is separate from DSP (can be replaced by hardware sequencer)
- No pfft~ or FFT in core (problematic on embedded)

To port to RNBO: export gen~ codebox → RNBO patch → VST/AU or embedded target.
To port to hardware: gen~ → RNBO → Daisy/Bela/custom DSP board.
